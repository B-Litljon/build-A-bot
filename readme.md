# Build-A-Bot: Universal Scalper V3.4

An advanced Machine Learning trading system for Alpaca, specializing in high-frequency scalping across Crypto and Equities. The system utilizes a dual-stage Meta-Labeling architecture (Angel/Devil) and is designed for fully autonomous operation with integrated DevOps retraining loops.

## Core Architecture: Gen-3 "Universal Scalper"

Build-A-Bot has evolved into a sophisticated ML engine that moves beyond simple rule-based strategies to a predictive, multi-stage inference pipeline.

### Key Components

*   **LiveOrchestrator:** A dual-stream engine that multiplexes Equities (IEX) and Crypto (Alpaca) data into a single asyncio event loop. It manages the complete lifecycle of a trade through a robust state machine (`FLAT`, `PENDING`, `IN_TRADE`, `COOLING`).
*   **Angel/Devil Meta-Labeling:** A two-stage Random Forest pipeline:
    *   **The Angel (Stage 1):** Proposes trades based on 18-feature technical indicators (High Recall).
    *   **The Devil (Stage 2):** Filters proposals for high-conviction entries, trained on a "Survival Target" to maximize profit factor (High Precision).
*   **Universal Watchdog:** A tick-by-tick software watchdog that monitors all open positions once per second, enforcing ATR-based dynamic Stop-Loss and Take-Profit brackets.
*   **The Cure V2:** An automated DevOps pipeline that monitors model drift, triggers retraining on new market regimes, and promotes models only after passing rigorous validation gates.

## Features

- **18-Feature Vector:** Includes standard technicals, Higher-Timeframe (5m) trend agreement, and Phase 5 Microstructure features (Range Coiling, Wick Analysis).
- **ATR-Based Dynamic Brackets:** Exits are scaled automatically based on current market volatility (NATR-14).
- **Cold/Warm Feature Path:** Optimized inference that avoids expensive HTF recomputes on every 1-minute bar.
- **Rich CLI Dashboard:** Real-time monitoring of symbol states, convictions, and system activity.
- **Zero-Latency State Sync:** Automatically adopts and manages open positions upon restart.

## Getting Started

### Prerequisites

- Python 3.12+
- [TA-Lib C library](https://ta-lib.org)
- Alpaca API Keys (Paper or Live)

### Installation

1.  **Clone and Setup:**
    ```bash
    git clone <repo-url>
    cd build-A-bot
    pip install pipenv && pipenv install && pipenv shell
    ```

2.  **Configuration:**
    Populate your `.env` file:
    ```
    alpaca_key=YOUR_KEY
    alpaca_secret=YOUR_SECRET
    DISCORD_WEBHOOK_URL=YOUR_WEBHOOK (optional)
    ```

### Running the Bot

```bash
python run_live.py
```
For headless production deployment:
```bash
python run_live.py --daemon
```

## DevOps & Retraining

The system is designed to be self-healing. Use the provided pipeline to validate or retrain models:

```bash
bash run_pipeline.sh
```
This orchestrates Data Harvesting -> Replay Simulation -> Performance Evaluation -> Automated Retraining (The Cure V2).
